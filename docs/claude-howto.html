4. コンテキスト管理

最重要ポイント - これを理解しないと使いこなせない

コンテキストウィンドウとは

Claudeが「覚えていられる」情報量には上限がある

コンテキストの消費

📄 ファイル読み込み

1ファイル読むたびに数千トークン消費。大きなファイルは特に注意。

💬 会話の蓄積

やり取りが長くなるほど消費。エラーメッセージも含む。

🔧 ツール実行結果

コマンド出力やログも全部コンテキストに入る。

📚 システム情報

CLAUDE.mdやスキル定義も常に読み込まれている。

コンテキストが溢れると...

  • 指示を忘れる - 最初に言ったことを無視し始める
  • 同じミスを繰り返す - 修正しても元に戻る
  • 的外れな回答 - 文脈を見失う
  • 無限ループ - 同じ操作を繰り返す

コンテキスト使用量の確認方法

/statusline でターミナル下部にリアルタイム表示

[Opus] Context: 42.5% │ $0.12 │ 📁 my-project
  • Context % - 使用率を常に把握
  • 50%超 → そろそろ注意
  • 80%超 → /clear を検討

対策① /clear でリセット

> /clear
✓ Context cleared. Starting fresh conversation.

いつ使う？

  • タスクが一区切りついたとき
  • 話題を大きく変えるとき
  • 動作がおかしくなってきたとき

/clear vs /compact

状況に応じて使い分けよう

/clear /compact
履歴 完全にリセット 要約して圧縮
文脈 全部消える 圧縮されて残る
用途 完全に仕切り直したい時 トークン節約しつつ文脈キープ
オプション なし フォーカス指示を付けられる
> /compact 認証モジュールに集中して
✓ Conversation compacted with focus on authentication module

長時間作業でトークン節約したいけど文脈は残したい → /compact がおすすめ

5. サブエージェント

重い処理を委譲してコンテキストを節約

サブエージェントで委譲

重い調査はサブエージェントに任せて、結果だけ受け取る

組み込みサブエージェント

Claude Code には複数の組み込みサブエージェントがある（主要なもの）

名前 モデル 用途
Explore デフォルトでHaiku（高速） 読み取り専用。ファイル検索、コード調査
Plan 継承 Plan Mode時の情報収集
General-purpose 継承 複雑なマルチステップタスク

他にも Bash、statusline-setup、Claude Code Guide など特定タスク用のヘルパーエージェントも存在

Exploreはデフォルトで Haiku を使う = 高速かつ低コストで大量のファイルを調査できる

サブエージェントのメリット

🧠 コンテキスト保全

調査結果がメインの会話を汚さない。要約だけ受け取る。

🔒 ツール制限

サブエージェントごとに使えるツールを制限できる。安全性向上。

💰 コスト管理

軽いタスクはHaikuに任せてコスト削減。

⚡ 並列実行

複数のサブエージェントを同時に走らせて高速化。

サブエージェントの実行モード

フォアグラウンド

  • 完了まで待機
  • 権限確認が対話的に可能
  • 質問も受け付ける

バックグラウンド

  • 並行して作業継続
  • 事前に権限を確認
  • Ctrl+B で切り替え

「バックグラウンドで実行して」と指示するか、実行中に Ctrl+B を押す

カスタムサブエージェントの作成

> /agents
1. View all available subagents
2. Create new agent
3. Edit existing agent

配置場所

  • ~/.claude/agents/
    全プロジェクトで利用可
  • .claude/agents/
    このプロジェクトのみ

ファイル形式

Markdownファイル（YAML frontmatter + システムプロンプト）

カスタムサブエージェントの例

# ~/.claude/agents/code-reviewer.md
---
name: code-reviewer
description: コード変更後に自動でレビュー
tools: Read, Grep, Glob, Bash
model: sonnet
---
あなたはシニアコードレビュアーです。
コード品質、セキュリティ、ベストプラクティスを確認してください。
  • description が重要 - Claudeが自動で使うかの判断基準
  • tools で使えるツールを制限
  • model で使用モデルを指定

サブエージェントの活用パターン

📊 大量出力の隔離

テスト実行、ログ処理など大量の出力をサブエージェントに閉じ込める

「サブエージェントでテスト実行して、失敗だけ報告して」

🔀 並列リサーチ

独立した調査を複数同時に実行

「認証、DB、APIモジュールを並列で調査して」

🔗 チェーン実行

サブエージェントを連続で呼び出す

「reviewerで問題を見つけて、fixerで修正して」

🔄 レジューム

途中で止まったサブエージェントを再開

「さっきのレビューを続けて」

6. Slash Command / Skill / Plugin

カスタマイズと拡張の仕組み

用語の整理

Slash Command

/ で呼び出すコマンド

組み込み: /clear, /help, /compact など

Skill

ユーザーが作るカスタムコマンド

自動起動も可能。.claude/skills/ に配置

Plugin

配布可能なパッケージ

Skills, Agents, Hooks, MCPをまとめて配布

Marketplace

Pluginの公開・共有場所

GitHubリポジトリをストアとして登録

Skill とは

ユーザー作成のカスタムコマンド

.claude/skills/ に配置

  • 自動起動 - Claudeが判断して適用
  • 手動呼び出し - /skill名
  • ディレクトリ構造 - 複数ファイル対応
# 手動で呼び出し
> /code-review
# または自動起動
> このコードをレビューして
[code-review skill activated]

Skillの作り方

# .claude/skills/code-review/SKILL.md
---
name: code-review
description: コードをレビューする
---
セキュリティとパフォーマンスを
チェックしてください。

description が超重要

Claudeが自動起動するかどうかの判断基準になる

配置場所

  • ~/.claude/skills/ 全プロジェクト
  • .claude/skills/ このプロジェクトのみ

対話形式で作成できる skill-creator もあり（ハンズオンで体験）

Plugin = 配布パッケージ
